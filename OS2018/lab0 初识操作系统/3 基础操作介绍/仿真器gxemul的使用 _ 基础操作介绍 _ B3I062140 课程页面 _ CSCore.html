选项。</p> <div class="sourceCode" linenos=""> <pre class="sourceCode bash" style="padding-left: 90px;"><code class="sourceCode bash"><span class="kw">ls</span> - list directory contents 用法:<span class="kw">ls</span> [选项]... &nbsp; &nbsp; &nbsp;那么目录非空时怎么办呢？这就需要用到搞破坏者最喜欢的 rm 命令了，rm 命令可以删除一个目录中的一个或多个文件或目录，也可以将某个目录及其下属的所有文件及其子目录均删除掉。对于链接文件，只是删除整个链接文件，而原有文件保持不变。</p> <div class="sourceCode" linenos=""> <pre class="sourceCode bash" style="padding-left: 90px;"><code class="sourceCode bash"><span class="kw">rm</span> - remove files or directories 用法:<span class="kw">rm</span> [选项]... 文件... 选项（常用）： <span class="kw">-r</span> 递归删除目录及其内容 <span class="kw">-f</span> 强制删除。忽略不存在的文件，不提示确认</code></pre> </div> <p>&nbsp; &nbsp; &nbsp; &nbsp;使用 rm 命令要格外小心，因为一旦删除了一个文件，就无法再恢复它。所以，在删除文件之前，最好再看一下文件的内容，确定是否真要删除。</p> <p>&nbsp; &nbsp; &nbsp; &nbsp;此外，rm 命令可以用 -i 选项，这个选项在使用文件扩展名字符删除多个文件时特别有用。使用这个选项，系统会要求你逐一确定是否要删除。这时，必须输入 y 并按 Enter 键，才能删除文件。如果仅按 Enter 键或其他字符，文件不会被删除。与之相对应的就是 -f 选项，强制删除文件或目录，不询问用户。使用该选项配合 -r 选项，进行递归强制删除，强制将指定目录下的所有文件与子目录一并删除可以达到毁灭性效果。例如：rm &ndash;rf / 即可强制递归删除全盘文件（但是需要一定权限）。</p> <p>&nbsp; &nbsp; &nbsp; &nbsp;在需要键入文件名/目录名时，可以使用 TAB 键补足全名，当有多种补足可能时双击 TAB 键可以显示所有可能选项。</p> <p>&nbsp; &nbsp; &nbsp; &nbsp;学会创建目录后，我们就需要进入我们新建的目录中创建和修改文件来完成工作了。使用 cd 命令来切换工作目录至dirname，其中dirname可为绝对路径或相对路径。若目录名称省略，则变换至使用者的主目录home (等价于cd &nbsp; )。另外，在Linux文件系统中， . 表示当前所在目录， .. 表示当前目录位置的上一层目录，同学们在使用ls &ndash;a后看到的目录中的 . 和 .. 即是如此。</p> <div class="sourceCode" linenos=""> <pre class="sourceCode bash" style="padding-left: 90px;"><code class="sourceCode bash"><span class="kw">cd</span> - change working directory 用法:<span class="kw">cd</span> [路径] <span class="kw">e.g.</span> <span class="kw">cd</span> . 变更至当前目录 <span class="kw">cd</span> .. 变更至父目录</code></pre> </div> <p>&nbsp; 只在/var/lib/slocate资料库中找。locate的速度比find快，它并不是真的查找文件，而是查数据库，所以locate的查找并不是实时的，而是以数据库的更新为准，一般是系统自己维护，也可以手工升级数据库。</span></p> <pre> locate&nbsp;</pre> <pre> 用法 locate [选项] 文件名</pre> <p>&nbsp; Linux的文件调用权限分为三级 : 文件拥有者、群组、其他。利用 chmod 可以藉以控制文件如何被他人所调用。</p> <pre> chmod 权限设定字串 文件...</pre> <pre> 权限设定字串格式 :</pre> <pre> [ugoa...][[+-=][rwxX]...][,...]</pre> <p>其中：</p> <p>u 表示该文件的拥有者，g <p>abc为三个数字，分别表示拥有者，群组，其他人的权限。r=4，w=2，x=1，用这些数字的加和来表示权限。例如</p> <p>chmod 777 file和chmod a=rwx file效果相同。</p> <p>diff命令用于比较文件的差异。</p> <pre> diff [选项] 文件1 文件2</pre> <pre> 常用选项</pre> <pre> -b &nbsp;&nbsp;</pre> <pre> 选项(常用):</pre> <pre> -n:使用安静模式。在一般sed的用法中，输入文本的所有内容都会被输出。加上-n参数后，则只有经过sed 处理的内容才会被显示。</pre> <pre> -e: 进行多项编辑，即对输入行应用多条sed命令时使用。</pre> <pre> my.txt&nbsp;</pre> <p>删除my.txt文件的第二行到最后一行。</p> <pre> sed '2,$d' my.txt&nbsp;</pre> <p>在整行范围内把str1替换为str2。如果没有g标记，则只有每行第一个匹配的str1被替换成str2。</p> <pre> sed 's/str1/str2/g' my.txt</pre> <p>&nbsp;-e选项允许在同一行里执行多条命令。例子的第一条是第四行后添加一个str，第二个命令是将str替换为aaa。命令的执行顺序对结果有影响。</p> <pre> sed -e '4a\str ' -e 's/str/aaa/' my.txt&nbsp;</pre> <p></p> <p>awk是一种处理文本文件的语言，是一个强大的文本分析工具。这里只举几个简单的例子，学有余力的同学可以自行深入学习。</p> <pre> awk '$1&gt;2 {print $1,$3}' my.txt</pre> <p>这个命令的格式为awk 'pattern action' file，pattern为条件，action为命令，file为文件。命令中出项的$n代表每一行中用空格分隔后的第n项。所以该命令的意义是文件my.txt中所有第一项大于2的行，输出第一项和第三项。</p> <pre> awk -F, '{print $2}'&nbsp; my.txt</pre> <p>-F选项用来指定用于分隔的字符，默认是空格。所以该命令的$n就是用，分隔的第n项了。</p> <p>首先创建一个文件my.sh，向其中写入一下内容(写入内容可以查看后面的vim和nano教学)：</p> <pre> #!/bin/bash</pre> <pre> #balabala</pre> <pre> echo "Hello World!"</pre> <p>我们自己创建的shell脚本一般是不能直接运行的，需要添加运行权限:</p> <pre> chmod +x my.sh</pre> <p>添加权限之后，我们可以使用bash命令来运行这个文件，把我们的文件作为它的参数：</p> <pre> bash my.sh</pre> <p>或者使用之前介绍的指令source:</p> <pre> source my.sh</pre> <p>这样有些不方便，一种更方便的用法是：</p> <pre> ./my.sh</pre> <p>在脚本中我们通常会加入#!/bin/bash到文件首行，以保证我们的脚本默认会使用bash。第二行的内容是注释，以#开头。第三行是命令。</p> <p></p> <p>重定向和管道</p> <p>这部分我们将学习如何实现linux命令的输入输出怎样定向到文件，以及如何将多个指令组合实现更强大的功能。</p> <p>shell使用三种流：</p> <pre> 标准输入：stdin ，由0表示</pre> <pre> 标准输出：stdout，由1表示</pre> <pre> 标准错误：stderr，由2表示</pre> <p>重定向和管道可以重定向以上的流。</p> <p>重定向在前面已经有过介绍，这里只做一点补充。</p> <p>2&gt;&gt;可以将标准错误重定向。三种流可以同时重定向，举例:</p> <pre> command &lt; input.txt&nbsp;1&gt;output.txt 2&gt;err.txt</pre> <p>管道：</p> <p>管道符号&ldquo;|&rdquo;可以连接命令：</p> <pre> command1 | command2 | command3 | ...</pre> <p>以上内容是将command1的stdout发给command2的stdin，command2的stdout发给command3的stdin，依此类推。</p> <p>举例:</p> <pre> cat my.sh | grep "Hello"</pre> <p>上述命令的功能为将my.sh的内容输出给grep指令，grep在其中查找字符串。</p> <p>&nbsp; &nbsp; &nbsp; &nbsp;&nbsp;<span style="font-family: monospace, serif; white-space: pre-wrap;">cat &lt; my.sh | grep "Hello" &gt; output.txt"</span><span style="font-size: 1em;">&nbsp;</span></p> <p>上述命令重定向和管道混合使用，功能为将my.sh的内容作为cat指令参数，cat指令stdout发给grep指令的stdin，grep在其中查找字符串，最后将结果输出到output.txt。</p> <p>shell传递参数与函数</p> <p>我们可以向shell脚本传递参数。</p> <p>my2.sh的内容</p> <pre> echo $1</pre> <p>执行命令</p> <pre> ./my2.sh msg</pre> <p>则shell会执行echo msg这条指令。</p> <p>$n就代表第几个参数，而$0也就是命令，在例子中就是./my2.sh。</p> <p>除此之外还有一些可能有用的符号组合</p> <p>$# 传递的参数个数</p> <p>$* 一个字符串显示传递的全部参数</p> <p>shell中的函数也用类似的方式传递参数。</p> <pre> function 函数名&nbsp;()</pre> <pre> {</pre> <pre> commands</pre> <pre> [return int]</pre> <pre> }</pre> <p>function或者()可以省略其中一个。</p> <p>举例</p> <pre> fun(){</pre> <pre> echo $1</pre> <pre> echo $2</pre> <pre> echo &ldquo;the number of parameters is $#&rdquo;</pre> <pre> }</pre> <pre> fun 1 2</pre> <p></p> <p>shell流程控制</p> <p>shell脚本中也可以使用分支和循环语句。学有余力的同学可以学习一下。</p> <p>if的格式：</p> <pre> if condition</pre> <pre> then</pre> <pre> command1&nbsp;</pre> <pre> command2</pre> <pre> ...</pre> <pre> fi</pre> <p>或者写到一行</p> <pre> if condition; then command1; command2; ... fi</pre> <p>举例</p> <pre> a=1</pre> <pre> if [ $a -ne 1 ]; then echo ok; fi</pre> <p>条件部分可能会让同学们感到疑惑。[]是一个判断符号，条件语句要放在[]里面，并且前后两边必须要有空格<span color="#4d4d4d" face="Microsoft YaHei, SF Pro Display, Roboto, Noto, Arial, PingFang SC, sans-serif" style="color: #4d4d4d; font-family: 'Microsoft YaHei', 'SF Pro Display', Roboto, Noto, Arial, 'PingFang SC', sans-serif;"><b>。</b></span>中括号包含的条件表达式的-ne是关系运算符，它们和c语言的比较运算符对应如下。</p> <pre> -eq&nbsp; ==(equal)</pre> <pre> -ne&nbsp; !=(not equal)</pre> <pre> -gt&nbsp; &gt;(greater than)</pre> <pre> -lt&nbsp; &lt;(less than)</pre> <pre> -ge&nbsp; &gt;=(greater or equal)</pre> <pre> -le&nbsp; &lt;=(less or equal)</pre> <p>条件也可以写true或false。</p> <p>变量除了使用自己定义的以外，还有一个比较常用的是$?，代表上一个命令的返回值。比如刚执行完diff后，若两文件相同$?为0。</p> <p>实际上condition的位置上也是命令，当返回值为0时执行。左中括号是指令，$a、-ne、1、]是指令的选项，关系成立返回0。true则是直接返回0。condition也可以用diff file1 file2来填。</p> <p>while语句格式如下</p> <pre> while condition</pre> <pre> do</pre> <pre> commands</pre> <pre> done</pre> <p>while语句可以使用continue和break这两个循环控制语句。</p> <p>例如创建10个目录，名字是file1到file9</p> <pre> a=1</pre> <pre> while [ $a -ne 10 ]</pre> <pre> do</pre> <pre> mkdir file$a</pre> <pre> a=$[$a+1]</pre> <pre> done</pre> <p>有两点请注意：流程控制的内容不可为空。运算符和变量之间要有空格</p> <p>除了以上内容，shell还有for，case，else语句，逻辑运算符等语法，内容有兴趣的同学可以自行了解。</p> <p></p> </div> </div> <div class="vert vert-1" 仿真器gxemul的使用

gxemul是我们运行小操作系统的仿真器，它可以帮助我们运行和调试小操作系统。直接输入gxemul会显示帮助信息。

gxemul运行选项:

    -E 仿真机器的类型
    -C 仿真cpu的类型
    -M 仿真的内存大小
    -V 进入调试模式

举例：

    gxemul –E testmips -C R3000 –M 64 vmlinux    用gxemul运行vmlinux

    gxemul –E testmips -C R3000 –M 64 –V vmlinux    以调试模式打开gxemul，对vmlinux进行调试（进入后直接中断，输入continue或step才会继续运行，在此之前可以进行添加断点等操作）

进入gxemul后使用Ctrl-C可以中断运行。中断后可以进行单步调试，执行如下指令：

    breakpoint add addr添加断点
    continue 继续执行
    step [n] 向后执行n条汇编指令
    lookup name|addr 通过名字或地址查找标识符
    dump [addr [endaddr]] 查询指定地址的内容
    reg [cpuid][,c] 查看寄存器内容，添加”,c”可以查看协处理器
    help 显示各个指令的作用与用法
    quit 退出

(以上中括号表示内容可以没有)

更多gxemul相关的信息参考http://gavare.se/gxemul/gxemul-stable/doc/index.html
